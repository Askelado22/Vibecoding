# GGSEL Разбор товаров

Полноценное приложение (Next.js + Prisma + SQLite) для ускоренного разбора товаров с интеграцией Google Sheets (через GAS endpoint). Проект состоит из трёх вкладок: рабочей карточки, общего списка и админ-панели.

## Возможности

- Аутентификация по email/паролю с ролями `admin` и `worker` (JWT в HttpOnly cookie).
- Рабочее место сотрудника с очередью только его товаров, быстрым редактированием статуса и хлебных крошек, подсказками и интегрированным скрейпером GGSEL.
- Табличный общий список с фильтрами, поиском и управлением назначением товаров.
- Админ-панель с метриками, загрузкой подсказок, управлением пользователями и контролем синхронизации с Google Sheets.
- Фоновая двунаправленная синхронизация с таблицей через GAS (ручная и каждые 30 минут при включённом авто-синке).

## Быстрый старт

```bash
npm install
npm run prisma:migrate
npm run db:seed
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000). Для входа используйте данные из сидов:

- Admin: `admin@example.com` / `password123`
- Worker: `worker@example.com` / `worker123`

## Переменные окружения

Скопируйте файл `.env.example` в `.env` и укажите значения:

| Переменная        | Назначение                                                                 |
| ----------------- | -------------------------------------------------------------------------- |
| `DATA_SOURCE`     | Всегда `db` — UI работает с локальной БД.                                  |
| `DATABASE_URL`    | Путь к SQLite файлу (`file:./dev.db`).                                     |
| `GAS_BASE_URL`    | URL опубликованного GAS-скрипта (веб-приложение).                          |
| `SHEET_RANGE`     | Диапазон вида `Лист1!A:M`.                                                 |
| `JWT_SECRET`      | Секрет для подписи JWT.                                                    |
| `TZ`              | Часовой пояс. Используйте `Europe/Moscow`.                                |
| `AUTO_SYNC_ENABLED` | Первичное состояние авто-синхронизации при создании настроек.            |

## Настройка Google Apps Script

1. Создайте Google Apps Script, подключенный к нужной таблице.
2. Реализуйте обработку запросов `GET` и `POST`:
   - `GET ?action=pull&range=Лист1!A:M` — должен возвращать `{ "rows": [[...A..M], ...] }`.
   - `POST` с телом `{ action: 'push', range: 'Лист1!A:M', rows: [[...A..M], ...] }` — обновляет/вставляет строки.
3. Опубликуйте скрипт как веб-приложение с доступом «Anyone with the link». URL поместите в `GAS_BASE_URL`.

## Синхронизация

- Ручной запуск: POST `/api/admin/sync/run` (кнопка в админке).
- Авто-синк: каждые 30 минут при включённом флаге (toggle в админке). Логи выводятся в консоль сервера.
- Алгоритм «последняя запись побеждает» сравнивает `updated_at` (максимум из E/H/K).
- Push в таблицу передаёт локальные изменения по полям A..M (кроме L — только чтение).

## Скрейпер GGSEL

Эндпоинт `/api/fetchProduct?url=...` делает серверный запрос к странице товара, парсит название, описание, цену, картинки и хлебные крошки, кэшируя результат на 15 минут.

## Подсказки хлебных крошек

В админке можно загрузить CSV или JSON:

- CSV: `keyword,path,score,description` (заголовок необязателен).
- JSON: массив объектов `{ "titleMatch": "steam", "path": "Игры > Steam", "score": 2 }`.

После загрузки сервис подсказок выдаёт до 5 вариантов с максимальными скорингами.

## Запуск в production

```
npm run build
npm run start
```

Перед деплоем выполните `npm run prisma:deploy` для применения миграций и `npm run db:seed` (по необходимости).

## Дополнительно

- Тема — тёмная с акцентами blue/pink, TailwindCSS.
- Логика аудита: изменения статуса и хлебных крошек фиксируют автора/время, «Завершил» заполняет I/J/K и `is_completed`.
- Оптимистичные блокировки реализованы через `updated_at` (конфликт -> сообщение).
