# XML Breadcrumbs to XLSX Converter

Этот репозиторий содержит CLI-скрипт `xml_breadcrumbs_to_xlsx.py`, который
потоково парсит крупный XML-файл с элементами `<item>` и формирует XLSX-таблицу
с хлебными крошками. Скрипт специально рассчитан на работу с большими файлами,
поэтому использует `xml.etree.ElementTree.iterparse` и режим "write-only" при
создании XLSX-файла.

## Требования

- Python 3.8 или новее
- Установленный пакет [`openpyxl`](https://openpyxl.readthedocs.io/)
- Доступ к исходному XML-файлу с элементами `<item>`

## Обзор архитектуры скрипта

Ниже приведён детальный разбор ключевых функций скрипта.

### `Item`
`@dataclass`, описывающий элемент каталога. Хранит `item_id`, `parent_id`,
`digi_id` и нормализованное имя для сегмента хлебных крошек.

### `parse_items(xml_path: Path) -> Tuple[Dict[int, Item], List[int]]`
- Потоково читает XML-файл и обрабатывает только теги `<item>`.
- Извлекает значения `<id>`, `<parent_id>`, `<name>` и `<digi_catalog>` (при
  пустом `<name>` подставляет `<title>`).
- Преобразует идентификаторы в `int`, обрабатывает `0`, `null`, пустые строки.
- Сохраняет элементы в словаре и возвращает список идентификаторов в порядке
  появления, чтобы сохранить исходную последовательность.

### `build_breadcrumbs(...) -> Iterable[Tuple[str, Optional[int], int]]`
- Рекурсивно восстанавливает цепочку родителей для каждого элемента.
- Кэширует уже построенные цепочки, чтобы избегать повторных вычислений.
- Позволяет настроить разделитель сегментов и плейсхолдер для пустых имён.
- Отсекает записи без `digi_id`, чтобы в итоговом файле не появлялись пустые
  значения.
- Обнаруживает циклы в структуре (если элемент ссылается на себя или
  возникает петля).

### `write_xlsx(rows, output_path) -> int`
- Проверяет наличие зависимости `openpyxl`.
- Создаёт книгу в режиме `write_only=True`, добавляет заголовок и построчно
  записывает данные, не накапливая их в памяти.
- Возвращает количество записанных строк (без учёта заголовка).

### `build_arg_parser()`
- Конфигурирует CLI: входной XML, путь к XLSX, разделитель, плейсхолдер пустых
  имён, уровень логирования.

### `configure_logging(level)`
- Настраивает базовую конфигурацию логирования согласно выбранному уровню.

### `main(argv=None) -> int`
- Считывает аргументы командной строки.
- Настраивает логирование, запускает разбор XML, построение хлебных крошек и
  сохранение результата.
- Возвращает код выхода `0` при успехе, выбрасывает исключения при критических
  ошибках (например, отсутствует `openpyxl`).

## Пошаговый гайд по запуску (PowerShell)

Ниже приведён сценарий, рассчитанный на Windows PowerShell. Каждая команда
описана и приведена в рекомендуемом порядке.

1. **Создайте и активируйте виртуальное окружение** (рекомендуется, чтобы не
   смешивать зависимости):

   ```powershell
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1
   ```

2. **Обновите `pip` (опционально, но полезно для корректной установки
   пакетов):**

   ```powershell
   python -m pip install --upgrade pip
   ```

3. **Установите зависимость `openpyxl`:**

   ```powershell
   pip install openpyxl
   ```

4. **Запустите конвертацию XML → XLSX:**

   ```powershell
   python .\xml_breadcrumbs_to_xlsx.py `
       path\to\source.xml `
       path\to\result.xlsx `
       --separator " > " `
       --empty-name-placeholder "[id {id}]" `
       --log-level INFO
   ```

   - `path\to\source.xml` — путь к исходному XML-файлу.
   - `path\to\result.xlsx` — путь к целевому XLSX.
   - `--separator` — необязательный параметр, задаёт строку между сегментами
     хлебных крошек (по умолчанию `" > "`, как в примере заказчика).
   - `--empty-name-placeholder` — шаблон, который будет подставлен, если поле
     `<name>` пусто. Можно использовать `{id}` для подстановки идентификатора.
   - `--log-level` — уровень логирования (`CRITICAL`, `ERROR`, `WARNING`,
     `INFO`, `DEBUG`).

5. **(Опционально) Проверьте, что скрипт компилируется без синтаксических
   ошибок:**

   ```powershell
   python -m compileall .\xml_breadcrumbs_to_xlsx.py
   ```

После завершения работы скрипта в указанном XLSX-файле появится лист
`breadcrumbs` с колонками `breadcrumbs`, `digi_id`, `id`.

## Обработка ошибок и рекомендации

- Если в XML встречаются элементы с повторяющимися `<id>`, скрипт оставит
  последний встреченный элемент и предупредит через логирование уровня
  `WARNING`.
- При отсутствии `openpyxl` или при повреждённом XML выполнение остановится с
  понятным сообщением и рекомендацией, что нужно исправить.
- Скрипт регистрирует прогресс: чтение XML, количество элементов, запись XLSX и
  итоговое число строк. Используйте `--log-level DEBUG`, чтобы увидеть
  дополнительные сообщения о пропущенных элементах (например, без `digi_id`) и
  проблемах с преобразованием идентификаторов.
